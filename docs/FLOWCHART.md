## 한 줄 요약

```text
웰컴 → 온보딩 → 로그인(카카오/구글/휴대폰)
  → 신원검증(동의 → OTP → 얼굴 → 미혼/보류)
  → 가입비 10만 안내(스킵 가능 → 매칭에서 미납이면 차단)
  → 프로필 → 가치관(5) → 라이프(5)
  → [운영] 남녀 매칭 제안 (2일 1건)
  → 양쪽 48h 수락 + 매칭비 3만(첫 1회 무료)
  → 불가능 시간·가능 장소(10일 창) → [운영] 일정 제안 → 수락
  → 만남 24h 전 채팅 오픈 → 만남 후 24h 내 종료
```

**하단 탭:** 매칭 · 일정 · 프로필  
**없음:** 자유 관심/맞관심, 스와이프, 번호 공개, 동성 매칭

## 1. 전체 마스터 플로우

```mermaid
flowchart TD
  W[웰컴] --> O[온보딩]
  O -->|시작하기| L[로그인<br/>카카오 / 구글 / 휴대폰]
  O -.->|이미 회원| L
  L --> V[신원검증 묶음]
  V -->|현재 혼인| X[가입 불가]
  V --> F[가입비 10만 안내<br/>스킵 가능]
  F --> P[프로필<br/>사진 필수 · 성별나이 자동채움]
  P --> VAL[가치관 5]
  VAL --> LIFE[라이프 5]
  LIFE --> MAIN[메인 탭]
  MAIN --> M[매칭 제안]
  M -->|미혼/가입비 미충족| BLUR[블러 + 안내]
  M -->|수락+결제 양쪽| S[일정 조율]
  S --> C[채팅 시간창]
```

## 2. 신원검증

```mermaid
flowchart TD
  A[0 약관·동의] --> B[1 휴대폰 OTP 실인증]
  B --> C[2 결과 표시<br/>이름·성별·나이·국적]
  C --> D[3~4 얼굴 실촬영]
  D --> E[5 미혼 인증]
  E -->|승인| OK[완료]
  E -->|현재 혼인| BLOCK[가입 불가]
  E -->|나중에| DEF[보류 · 매칭 불가]
  DEF --> OK
```

| 단계 | 핵심 규칙 |
|------|-----------|
| 동의 | 이용약관·개인정보·제3자·만50세 필수 / 마케팅 선택 |
| OTP | **실제 SMS**. 성공 시 이름·성별·나이·국적 저장 → 프로필 자동채움 |
| 얼굴 | **실제 카메라 본인확인** |
| 미혼 | 현재 혼인 = 가입 불가 / 과거 혼인+현재 미혼 OK / 나중에 = 매칭만 잠금 |

## 3. 가입비 · 기능 게이트

강제 리다이렉트 **없음**. 메인은 들어가고, 조건 미달 시 **해당 기능만** 막음.

```mermaid
flowchart LR
  A[A 로그인 직후] --> B[B 미혼 직후]
  B --> C[C 큐레이션 완료 후]
  C --> D[D 매칭 탭]
  D -->|미결제| LOCK[매칭 이용 불가<br/>블러]
```

| 미충족 | 결과 |
|--------|------|
| 가입비 미납 | 매칭 블러 (프로필·큐레이션은 가능) |
| 미혼 미승인 | 매칭 블러 |
| 프로필·가치관·라이프 미완 | 매칭·일정 등 핵심 막음 |
| 매칭 미성사 | 일정·채팅 본기능 없음 |

## 4. 매칭 (운영이 짝 지음)

```mermaid
flowchart TD
  OPS[운영: 남 1 + 여 1 선택] --> N[양쪽 알림: 매칭 제안]
  N --> T{48시간}
  T -->|수락| PAY[매칭비 3만<br/>첫 수락 1회 무료]
  PAY -->|성공| ME[내 수락·결제 완료]
  T -->|거절| RJ[거절]
  T -->|시간초과| EX[자동 취소]
  ME --> OT{상대}
  OT -->|대기| W[상대측 결제 대기중]
  OT -->|완료| OK[매칭 성사 → 일정]
  OT -->|거절/만료| CX[취소]
  RJ --> MSG[상대 문구:<br/>매칭비 미납으로 취소]
  EX --> MSG
  CX --> MSG
  MSG --> BAN[해당 쌍 영구 재매칭 금지]
```

| 규칙 | 내용 |
|------|------|
| 노출 | 한 번에 **1건**, **2일**에 다른 제안으로 갱신 |
| 짝 | **항상 남녀** |
| 후보 | 수도권 · 미혼 승인 · 가입비 완료 · 큐레이션 완료 · 블록 쌍 제외 · 유사도 보조 |
| 매칭비 | **양쪽 3만**, **제안 수락 시만** (일정 수락 시 추가 없음) |
| 거절 알림 | “거절당함” 대신 **미납 기한 취소** 문구 |

## 5. 일정 · 채팅

```mermaid
flowchart TD
  M[매칭 성사] --> U[유저: 불가능 시간 + 가능 장소 + 메모<br/>앞으로 10일 창]
  U --> O[운영: 날짜·시간·장소 제안]
  O --> A{양쪽 수락?}
  A -->|Yes| CF[일정 확정]
  A -->|No| WAIT[대기 / 운영 재제안]
  CF --> CH1[만남 24시간 전: 채팅 오픈]
  CH1 --> CH2[만남 후 24시간 내: 채팅 종료]
  CH2 --> TIP[안내: 만나서 연락처 교환]
```

| 규칙 | 내용 |
|------|------|
| 10일 | 제출 기한 아님 = **조율 날짜 범위** |
| 시간 입력 | **불가능 시간**을 고름 (가능 시간 아님) |
| 연락처 | 앱에서 **번호 비공개** |
| 채팅 | 윈도우 밖 송수신 불가 |

## 6. 결제

```mermaid
flowchart LR
  S[가입비 10만] -->|매칭 입장 전 필수| M
  M[매칭비 3만 × 양쪽] -->|첫 매칭 수락 1회 무료| OK[매칭 성사]
```

- 실제 PG 연동 (포트원 / 토스페이먼츠 등 — 개발자 추천 후 확정)
- 웹훅으로 결제 확정 후에만 수락·성사 처리
